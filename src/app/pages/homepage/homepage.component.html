To-Do List Application with Angular and TypeScript

Objective:

Create a dynamic/reactive to-do list application using Angular and TypeScript. This application should allow users to add, edit, delete, and mark tasks as completed. Additionally, the application should persist tasks in local storage and be optimized for performance.

Requirements:

Functionality:

  • Users should be able to add new tasks.
  • Users should be able to edit existing tasks.
  • Users should be able to delete tasks.
  • Users should be able to mark tasks as completed or uncompleted.
  • Persist the tasks in local storage after a page refresh.

Technical Specifications:

  • Use Angular for the front-end framework.
  • Use TypeScript to define component props and state, ensuring type safety.
  • Manage state effectively, using hooks or a state management library like NgRx.
  • Optimize the application to minimize unnecessary re-renders.

Performance Optimization:

  • Implement performance optimization techniques, such as lazy loading for components or virtualizing long lists if applicable.
  • Ensure that the application remains responsive and fast, even with a large number of tasks.

User Interface:

  • Provide a simple but user-friendly UI/UX.
  • Use CSS or a CSS-in-JS solution to style the application.

Code Quality:

  • Write clean, readable, and maintainable code.
  • Include comments and documentation where necessary.
  • Use TypeScript features effectively, such as interfaces and type definitions.

Bonus Points:

  • Implement a search or filter feature to allow users to find specific tasks.
  • Implement a TTL (time to live) for the tasks to persist.
  • Include unit tests for key components using a testing framework like Jest.
  • Implement drag-and-drop functionality to reorder tasks.

Submission Guidelines:

  • Provide a link to a GitHub repository with your completed application.
  • Include a README file with instructions on how to set up and run the application.
  • Ensure all dependencies are listed in the package.json file.
